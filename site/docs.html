B2B digital value reseller portal · two-repo architecture

Business model

Megasoft Market is a wholesale/reseller front-end. Partners log in, browse promotions and products, and submit purchases that credit digital value to end subscribers (mobile airtime, data bundles, prepaid SKUs — depending on upstream catalog).

It is not a cart-checkout e-commerce site for physical goods.

Architecture

Reseller (browser)
    ↓ JWT
megasoft-market (Vue 2 SPA)
    ↓ REST
megasoft-market-bff (Express)
    ↓ cached HTTP
Upstream digital value / DVS API
    ↓
async/transactions → credit subscriber

Repos

  • megasoft-market — Vue 2 client (promotions, products, purchase UI, admin)
  • megasoft-market-bff — Express BFF (auth, cache, proxy)

API routes (BFF)

  • POST /login · POST /token
  • GET /promotions · GET /products
  • POST /transactions — credit purchase
  • GET /transactions · GET /balances — admin only

Roles

  • Reseller — dashboard, browse, purchase/credit
  • Admin — transactions list, balance overview

Local dev

# BFF (port 8000) — needs .env with upstream API credentials
cd megasoft-market-bff && npm install && npm run dev

# Client (port 8080) — point VUE_APP_API_URL at BFF
cd megasoft-market && npm install && npm run serve

# Or Docker
npm run docker-build && npm run docker-run
